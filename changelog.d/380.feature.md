## Client-Driven OAuth Token Expiry

OAuth access tokens now support client-configurable expiry times, reducing authentication friction while maintaining security. Previously, all OAuth tokens expired after 1 hour, forcing users to re-authenticate multiple times per day.

Clients can now request custom token lifetimes by sending a `requested_expiry` parameter during OAuth token exchange. The CLI uses this to request 30-day tokens by default, reducing re-authentication from daily to monthly for typical workflows. The MCP server validates requests and enforces a configurable maximum limit (90 days by default) to prevent abuse. Web UI and MCP clients that don't specify custom expiry receive the default 1-day tokens, balancing convenience with security for interactive sessions.

Configure default and maximum token expiry using environment variables: `OAUTH_DEFAULT_TOKEN_TTL_SECONDS` (default: 86400 / 1 day) and `OAUTH_MAX_TOKEN_TTL_SECONDS` (default: 7776000 / 90 days). The server transparently honors client requests within these limits, with invalid or excessive values falling back to the default expiry.
