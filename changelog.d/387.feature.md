## Folder-Based Skill Loading

User-defined prompts repositories now support folder-based skills alongside flat `.md` files. Previously, skills requiring supporting files (shell scripts, manifests, templates) could only be distributed via git submodules, creating a disconnected distribution mechanism.

A directory containing a `SKILL.md` file is automatically discovered as a skill folder. The `SKILL.md` content becomes the prompt text, and all supporting files in the directory are base64-encoded and included in the REST API response via a new `files[]` field. Skill folders without supporting files (only `SKILL.md`) behave identically to flat `.md` prompts and are available through both MCP and REST. Skill folders with supporting files are excluded from MCP `prompts/list` and `prompts/get` entirely, since MCP clients cannot deliver the files needed for the skill to function.

Individual supporting files are limited to 5 MB before encoding. The skill name defaults to the directory name but can be overridden via frontmatter in `SKILL.md`. No new API endpoints are needed — the existing `POST /api/v1/prompts/:name` response includes the optional `files` field, maintaining full backward compatibility.
