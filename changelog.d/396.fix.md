## Fix: Dex Disabled by Default (Issue #396)

Fixes a crash when upgrading from v1.5.0 to v1.6.0 with `ingress.tls.enabled: false`. The MCP SDK requires HTTPS for OAuth issuer URLs (per RFC 8414), causing `Error: Issuer URL must be HTTPS` when the chart derived `http://` URLs for setups where TLS terminates at a reverse proxy or load balancer.

**Breaking change**: `dex.enabled` now defaults to `false` (was `true` in v1.6.0). OAuth authentication is opt-in — set `dex.enabled: true` when HTTPS is configured. Static token authentication (`DOT_AI_AUTH_TOKEN`) remains the default and works without HTTPS.

For setups with TLS termination at a reverse proxy, set `externalUrl` and `dex.externalUrl` explicitly. See the updated [TLS Configuration](https://devopstoolkit.ai/docs/mcp/setup/deployment#tls-configuration) docs. ([#396](https://github.com/vfarcic/dot-ai/issues/396))
